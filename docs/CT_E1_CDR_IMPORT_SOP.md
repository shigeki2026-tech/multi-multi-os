# CT-e1 通話呼詳細CSV → 応答率速報 SOP

## ゴール

CT-e1から取得した `通話呼詳細V3.5(CSV)` を、multi-multi-osの `応答率速報` で業務判断できる状態にする。

## 本線

1. CT-e1で人間が条件指定する。
2. `通話呼詳細V3.5(CSV)` を出力する。
3. V5保存ヘルパーでCSV保存を安定化する。
4. multi-multi-osの `応答率速報 > CDR取込` でCSVを取り込む。
5. `応答率速報 > 応答率閲覧` で、期間・業務グループ・個別回線・全回線・放棄秒を指定して確認する。

## 日常起動

保存ヘルパーは以下。

```powershell
cd "C:\Users\User\Documents\Projects\multi-multi-os-local\ct_e1_export_rpa"
.\bat\Start-CtE1CsvSaveHelper.bat
```

## 画面確認

`応答率速報 > 応答率閲覧` で確認する項目。

- 対象期間 開始
- 対象期間 終了
- 放棄秒
- 回線種別
  - 業務グループ
  - 個別回線
  - 全回線
- KPI
  - 入電数
  - 応答数
  - 呼損数
  - 応答率

## 除外番号

テストコール等の除外番号は、取込時に反映される。

後から除外番号を追加・変更した場合、既存集計には自動反映されない。必要なら対象期間の集計を削除し、同じCSVを再取込する。

## スケジュールタスク

旧 `CT-e1 Yoshikei Call Loss 18-05 / 20-05 / 21-05` は、応答率速報の更新経路ではない。

- 旧Call Lossタスクは `outbox txt/json` 系の呼損チェック。
- 応答率速報は `通話呼詳細V3.5(CSV)` のCDR取込と `answer_rate_threshold_stats` を使う。
- 現時点では旧Call Lossタスクを無効化しない。
