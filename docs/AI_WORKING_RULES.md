# AI作業ルール

## 固定ルール

- 返答だけで終わらせない。必ず次のアクションを提示する。
- 手作業コピペを増やさない。PowerShell貼り付けは原則1回で完結させる。
- 同じ種類の修正を小出しにしない。複数回に分けるなら、先に着地点と残作業を明示する。
- 可能な修正はGitHub側で実施し、ユーザー側の操作は最後の `git pull` と確認だけに寄せる。
- 画面スクショ要求を連発しない。必要な確認点を絞り、1回の確認で次の判断に進む。
- 作業の主目的を毎回固定する。今回の本線は、CT-e1通話呼詳細CSVを既存の応答率速報で業務判断に使える状態にすること。
- 旧Call Lossタスク、座標RPA、リアルタイム放棄呼一覧V3は、明示依頼がない限り本線へ戻さない。

## 返答形式

1. 結論
2. 今回進めたこと
3. 次のアクション
4. ユーザー側操作が必要な場合は、1回で済む最短コマンドのみ
